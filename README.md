# HealthTeam — магазин БАДов и витаминов (Telegram Mini App)

## Структура проекта
```
tg-shop/
  package.json       ← точка входа для npm
  .env.example        ← шаблон настроек (скопировать в .env)
  src/                ← backend (сервер)
  frontend/           ← страница приложения (открывается в Telegram)
```
Это ОДИН проект. Всё содержимое этой папки целиком загружается в GitHub как есть — ничего никуда отдельно вставлять не нужно, `frontend` уже лежит внутри на своём месте.

---

## Шаг 1. Создать бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather)
2. `/newbot` → придумайте имя и username (заканчивается на `bot`)
3. Сохраните **токен**, который пришлёт BotFather (вида `123456789:AAExample...`)
4. Узнайте свой Telegram ID и ID партнёра через [@userinfobot](https://t.me/userinfobot) (просто напишите ему)

## Шаг 2. Загрузить проект в GitHub

1. Зарегистрируйтесь на [github.com](https://github.com), если ещё нет аккаунта
2. Нажмите **New repository** (зелёная кнопка справа вверху) → назовите, например, `healthteam-shop` → **Create repository**
3. На странице пустого репозитория нажмите **uploading an existing file**
4. Перетащите туда ВСЕ файлы и папки из распакованного архива (`package.json`, `.env.example`, `src/`, `frontend/`, `.gitignore`) — можно просто перетащить всю папку целиком мышкой в окно браузера
5. Внизу нажмите **Commit changes**

Готово — код в GitHub.

## Шаг 3. Задеплоить на Railway (бесплатно для старта)

1. Зайдите на [railway.app](https://railway.app) → Sign in with GitHub
2. **New Project** → **Deploy from GitHub repo** → выберите ваш репозиторий
3. Railway сам определит, что это Node.js проект, и начнёт билд
4. Зайдите в **Variables** (переменные окружения) этого проекта и добавьте:
   ```
   BOT_TOKEN = ваш_токен_от_BotFather
   ADMIN_IDS = ваш_telegram_id,telegram_id_партнёра
   ```
5. Во вкладке **Settings** → **Networking** нажмите **Generate Domain** — Railway выдаст публичный адрес вида `https://your-app.up.railway.app`

Проверять локально не нужно — просто откройте этот адрес в браузере, должна открыться страница магазина.

## Шаг 4. Подключить Mini App к боту

1. В [@BotFather](https://t.me/BotFather): `/mybots` → выберите бота → **Bot Settings** → **Menu Button** → **Configure Menu Button**
2. Вставьте адрес из Railway (`https://your-app.up.railway.app`)
3. Введите текст кнопки, например «Открыть магазин»

Готово — у бота внизу появится кнопка, открывающая магазин.

## Постоянное хранилище данных (Railway Volume)

Данные (каталог, заказы, пользователи, бухгалтерия) хранятся в одном файле `data.json` — без внешней базы данных. Чтобы он не терялся при передеплоях, подключите Railway Volume:

1. В вашем проекте на Railway откройте сервис → вкладка **Settings** → раздел **Volumes**
2. **+ New Volume** → в поле пути монтирования укажите `/data` → Add
3. Во вкладке **Variables** добавьте переменную:
   ```
   DATA_DIR = /data
   ```
4. Railway передеплоит сервис — теперь `data.json` будет жить в подключённом томе и переживёт любые последующие обновления кода

Если переменную `DATA_DIR` не задавать — всё продолжит работать как раньше (данные хранятся в папке проекта), это подходит только для тестирования.

## Как быстро обновлять проект в будущем

Самый простой способ без консоли:
1. Заходите в свой репозиторий на GitHub
2. Открываете нужный файл → значок карандаша (Edit) → правите → **Commit changes**
3. Railway сам увидит изменение в GitHub и передеплоит проект за 30-60 секунд — ничего руками нажимать не надо

Если файлов много или меняется структура — проще всего каждый раз просить меня (Claude) внести изменения и присылать вам готовый обновлённый файл/архив, а вы через «Edit» или «Upload files» обновляете репозиторий.

## Как подключить меня к работе над проектом на постоянной основе

Есть два рабочих варианта:

**Вариант А — через этот чат (просто и без установки чего-либо)**
Присылайте мне файлы, которые нужно поменять, или просто описывайте задачу («добавь скидки», «добавь загрузку фото товара») — я меняю код и отдаю вам готовые файлы для загрузки в GitHub. Подходит, если правки происходят не каждый день.

**Вариант Б — Claude Code (для частой активной разработки)**
Это отдельное приложение от Anthropic, которое подключается напрямую к вашему GitHub-репозиторию на вашем компьютере: я смогу сам читать код, вносить правки и коммитить их в репозиторий — без пересылки файлов туда-сюда. Удобно, если планируете часто и быстро дорабатывать магазин (акции, новые разделы, интеграции).

Также можно один раз подключить GitHub-коннектор прямо в этом чате — тогда я смогу напрямую читать и обновлять ваш репозиторий из обычного чата, без Claude Code. Просто скажите — предложу подключение.
